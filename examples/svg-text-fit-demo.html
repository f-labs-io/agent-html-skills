Demonstrates the html-svg-diagrams rules: <foreignObject> for variable labels, plain <text> only for short fixed-length, edge labels with backing rects, ≥40px node spacing, ≥8px shape padding.

Three nodes, two edges. "Client" is fixed-length → plain <text>. "API Gateway / Authentication & Authorization Service" and "User Service / Profile read/write, preferences, account lifecycle" are variable / multi-line → <foreignObject>. Edge labels sit on small white rects so they stay readable if a path ever crosses them.

2. State machine with longer state names

State machine with mid-length state labels. Every state node uses <foreignObject> — the labels happen to be short, but the same pattern handles "Pending review" and an eventual "Pending senior-reviewer escalation" without code change. Edge labels (submit, approve(), reject()) have backing rects.

What this demonstrates

  • Short, fixed-length label → plain <text>. Box width is comfortably larger than label width × 8px + 32px padding.
  • Variable or potentially-long label → <foreignObject> + HTML <div>. Real word wrapping, real flex centering, real padding control. The label can grow without breaking the layout.
  • Edge labels sit on small white-fill <rect>s. If the path ever crosses another element, the label stays readable instead of merging with the line behind it.
  • Spacing ≥ 40px between any two nodes. Client→Gateway gap is 70px; Gateway→User Service gap is 70px; vertical state-machine arcs clear all box edges.
  • Padding ≥ 8px on every text-containing shape (used 10–16px throughout for breathing room).
